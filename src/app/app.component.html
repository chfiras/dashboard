Progess

More Info
ASSESSMENT *: STARTING POINT
  • Lack of knowledge on how many defects** each release produces
  • Don’t have in developer habits to follow code quality via using quality analysis tools (such as Sonar, Cast and SQI)
  • Lack of awareness on coding best practices (1000 top priority critical violations; 97% same bad practice)
  • Developers are not clearing all doubts when accepting a new requirement
HOW: STEPS
  • Launch software craft coaching program every quarter by selecting TOP 5 applications who have the most application incidents & projects in progress. The selection of application must be agreed with HIL, HOL, Domain Lead, Head of Domain IT
  • Software craft coaching program content:
  • Weekly grooming session 1 hour between BA & Developers with tech coach as facilitator: BA explain requirement / Developers explain the technical solution before coding
  • Taylor made coaching developer session 1 hour to review best practices & code . In this mandatory session 2 developers will be selected by the tech coach regarding the observation & interactions in the grooming session.
KEY RESULTS: TARGET
  • Implement defect indicator and decrease it
  • 8 developers confirmed by tech coach as a champion level 1 ” every quarter
  • 0 new technical debt ( 0 new top priority critical violation monitored via code analysis tool) incl. in OAT signoff
  • 4/5 feedback coaching program result by the ROTI (Return On Time Invested) in each session
